# IQ:NS 如何工作

## 从文档到结构化知识

标准、法规和机构规则以自然语言形式存在于 PDF 中。对人们逐页阅读很有用——但当你需要机器推理欧盟《人工智能法案》第13条如何与 NIST 控制或 ISO 条款相关时，就不太适用了。  
IQ:NS 将这些概念建模为形式本体——结构化词汇表，其中每个术语都有稳定的标识符、基于其来源的定义，以及与跨框架相关概念的明确关系。

## 四层结构

### 1. 概念建模

每个标准被拆分为其组成概念——义务、控制、风险类别、角色——并以 OWL 类的形式表示，配以 SKOS 标签和定义。每个概念都可追溯到其权威来源。

### 2. 跨框架对齐

框架之间不断重叠。GDPR 第22条和欧盟《人工智能法案》第13条都涉及透明度。ISO 42001 和 NIST AI RMF 都定义了风险管理流程。IQ:NS 使用 `skos:exactMatch`、`skos:broadMatch` 和 `skos:relatedMatch` 明确捕获这些对齐关系。  
一次查询即可展示框架的收敛点、分歧点以及存在的空白。

### 3. 情境画像

并非所有概念都适用于每种情境。IQ:NS 支持按司法管辖区、行业和能力类型进行画像——因此你可以查询“在欧盟的信用评分系统适用哪些规定？”并获得精确、去重的答案。

### 4. 持续维护

框架会变化。新标准会出现。社区维护并扩展本体，使知识保持最新。每个版本都有记录——你随时可以查看何时以及哪些内容发生了变化。

## 如何使用

- **在 GitHub 上浏览** — 下载 Turtle 文件，加载到你的三元组存储中
- **通过 SPARQL 查询** — 以编程方式跨框架提问
- **通过 MCP 连接** — 让你的 AI 代理直接对机构知识进行推理
- **与工具集成** — 接入 GRC 平台、数据目录、CI/CD 流水线

---

## IQ:NS 不是

- 不是策略引擎 — 它不执行规则
- 不是 GRC 平台 — 它不运行工作流或审批
- 不是咨询公司 — 决策由你做出；本体提供结构  

IQ:NS 是这些系统可以构建的 **语义基础**。
