# IQ:NS para Ingeniería de Software

## El desafío

Los equipos de ingeniería construyen y despliegan sistemas de IA pero a menudo carecen de claridad sobre lo que realmente significa el cumplimiento en el código — qué pruebas ejecutar, qué documentar, qué controles son relevantes para su sistema específico.

## Qué proporcionan las ontologías

- **Requisitos como datos estructurados** — obligaciones del marco consultables vía SPARQL, no enterradas en PDFs
- **Integración de canalizaciones** — incrustar consultas de ontología en CI/CD para mostrar requisitos relevantes en tiempo de compilación
- **Conocimiento listo para agentes** — el servidor MCP permite que los agentes de codificación IA razonen directamente sobre las reglas institucionales
- **Vocabulario compartido con cumplimiento** — mismos conceptos, mismas definiciones, sin sobrecarga de traducción

## Cómo encaja

Las ontologías se sitúan junto a tus herramientas existentes — Jira, CI/CD, registros de modelos. Proporcionan contexto estructurado para que las decisiones de ingeniería se conecten a los requisitos institucionales sin transferencias manuales.
