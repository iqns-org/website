# IQ:NS para Engenharia de Software

## O desafio

Equipes de engenharia constroem e implantam sistemas de IA, mas frequentemente carecem de clareza sobre o que a conformidade realmente significa no código — quais testes executar, o que documentar, quais controles são relevantes para seu sistema específico.

## O que as ontologias fornecem

- **Requisitos como dados estruturados** — obrigações do framework consultáveis via SPARQL, não enterradas em PDFs
- **Integração de pipeline** — incorpore consultas de ontologia no CI/CD para expor requisitos relevantes no momento da compilação
- **Conhecimento pronto para agentes** — o servidor MCP permite que agentes de codificação de IA raciocinem diretamente sobre regras institucionais
- **Vocabulário compartilhado com conformidade** — mesmos conceitos, mesmas definições, sem sobrecarga de tradução

## Como se encaixa

As ontologias ficam ao lado das suas ferramentas existentes — Jira, CI/CD, registros de modelos. Elas fornecem contexto estruturado para que decisões de engenharia se conectem aos requisitos institucionais sem transferências manuais.
