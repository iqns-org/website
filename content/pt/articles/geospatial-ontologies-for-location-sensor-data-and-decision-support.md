Data governance breaks at scale because "quality," "lineage," and "ownership" mean different things.

## Por que isso importa

Quando os conceitos de dados são padronizados, os engenheiros raciocinam sobre os dados de forma sistemática.

## Como isso se parece na prática

- A linhagem de dados é rastreável a partir de sistemas, humanos e pipelines de IA de forma consistente.
- Os limites de qualidade de dados significam a mesma coisa, seja aplicado por automação ou por humanos.
- Os metadados são reutilizáveis porque os esquemas descrevem os mesmos conceitos de forma consistente.

## Como as equipes o utilizam

- implementando governança de dados em nuvens, on-premises e armazenamentos de terceiros
- automatizando descoberta e classificação sem reconstruir a lógica de negócios
- comprovando a proveniência para auditoria, depuração e conformidade de forma sistemática

A linhagem de dados é auditável apenas quando os significados são consistentes desde a origem até o uso.
