# IQ:NS pour l'ingénierie logicielle

## Le défi

Les équipes d’ingénierie construisent et déploient des systèmes d’IA mais manquent souvent de clarté sur ce que signifie réellement la conformité dans le code — quels tests exécuter, quoi documenter, quels contrôles sont pertinents pour leur système spécifique.

## Ce que les ontologies offrent

- **Exigences sous forme de données structurées** — obligations du cadre interrogeables via SPARQL, pas enfouies dans des PDF  
- **Intégration au pipeline** — intégrer les requêtes d’ontologie dans CI/CD pour faire apparaître les exigences pertinentes lors de la construction  
- **Connaissance prête pour les agents** — le serveur MCP permet aux agents d’IA de coder de raisonner directement sur les règles institutionnelles  
- **Vocabulaire partagé avec la conformité** — mêmes concepts, mêmes définitions, aucune surcharge de traduction  

## Comment cela s'intègre

Les ontologies coexistent avec vos outils existants — Jira, CI/CD, registres de modèles. Elles fournissent un contexte structuré afin que les décisions d’ingénierie se connectent aux exigences institutionnelles sans transferts manuels.
